# AI短视频生成SOP：从选题到发布的全流程实战教程

> 如何用AI自动化生产高质量短视频？本文将揭秘完整的短视频创作SOP。

## 一、核心结论

**AI短视频生产的核心公式：**

```
高质量短视频 = 精准选题 × 标准化流程 × 自动化工具
```

经过实战验证，我们找到了一条高效、可复制的短视频生产路径：**从AI日报选题→标准化文案→TTS语音→Remotion渲染→视频号发布**，全流程可自动化执行。

---

## 二、关键成果

### 1. 选题评估标准

我们建立了一套科学的选题评估体系，5个维度量化打分：

| 维度 | 权重 | 说明 |
|------|------|------|
| 话题热度 | 30% | 搜索量、讨论量 |
| 新鲜度 | 20% | 发布时间越近越好 |
| 争议性/话题性 | 25% | 有无讨论空间 |
| 视觉化潜力 | 15% | 能否用短视频形式表达 |
| 受众广度 | 10% | 普通人能否看懂 |

**加分项：**
- 🇨🇳 中国公司动态（字节、阿里、腾讯）
- 🔥 国际巨头竞争（OpenAI、Google）
- 📹 视频/图像生成相关
- 💰 融资/估值动态
- 🤔 有争议性话题

### 2. 标准化SOP流程

```
AI日报选题 → 评估价值 → 整理文案(100-150字) → edge-tts生成语音 → 生成scenes-data → 用户确认 → remotion渲染 → 生成视频号标题/标签 → 输出
```

**Step 1：选题**
- 来源：每日AI新闻日报
- 评估：按5维度打分，选最高分

**Step 2：文案**
- 字数：100-150字
- 风格：口语化、新闻播报感
- 结构：开头抓眼球 → 核心内容 → 总结升华

**Step 3：TTS语音生成**
```bash
cd /root/.openclaw/workspace/ai-video-test
echo "你的文案" > public/script.txt
edge-tts -v "zh-CN-YunjianNeural" -f public/script.txt --write-media public/audio.mp3 --write-subtitles public/audio.srt
```

**Step 4：生成scenes-data**
- 解析SRT时间戳
- 配置关键词（中文≤3字）
- 匹配画面描述
- 发送确认

**Step 5：Remotion渲染**
```bash
npx remotion render Root.tsx MyVideo output.mp4
```

**Step 6：视频号标题与标签**
- 标题：1个，15-25字，视频号风格
- 标签：5-8个，#话题格式

## 三、底层原理

### 为什么是这套流程？

**第一性原理：短视频生产的本质是"信息封装"**

一条好的短视频，本质是将信息高效封装为受众可接受的视觉+听觉形式。

我们的SOP解决了3个核心问题：

1. **选题难** → 用AI日报+评估标准量化筛选
2. **制作慢** → 标准化流程减少决策损耗
3. **发布繁琐** → 自动化生成标题和标签

### 关键技术选型

| 环节 | 工具 | 选型理由 |
|------|------|----------|
| 语音合成 | edge-tts | 免费、质量高、中文自然 |
| 视频渲染 | Remotion | 代码化、可定制、自动化 |
| 字幕同步 | SRT解析 | 精确时间戳、适配Remotion |

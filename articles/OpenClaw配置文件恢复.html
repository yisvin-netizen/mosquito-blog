救命！我把OpenClaw配置文件删了——3种恢复方法+预防措施

用OpenClaw最崩溃的时刻是什么？我先说：不是AI不回答问题，而是某天早上醒来，发现openclaw.json打不开了。

先说结论：配置文件坏了别慌，3种方法帮你恢复。

为什么会坏？

常见原因就4种：

  1. 手动编辑格式错误 - JSON语法严格，一个逗号出错就报废
  2. 复制粘贴出问题 - 可能带入不可见字符
  3. 突然断电 - 写入被中断，文件不完整
  4. 覆盖性写入 - 没有保留备份

3种恢复方法

方法一：从备份恢复（最简单）

# 找到备份目录，通常在 ~/.openclaw/backups/
cp backup/openclaw-2026-02-20.json ~/.openclaw/config/openclaw.json
openclaw gateway restart

选择损坏前最近的备份文件即可。

方法二：重新配置（最稳妥）

  1. 备份当前坏文件
  2. 删除损坏的配置：rm ~/.openclaw/config/openclaw.json
  3. 重新初始化：openclaw gateway init
  4. 手动添加重要配置项
  5. 重启验证

方法三：从其他环境迁移（最省事）

  1. 在正常机器上找到配置文件
  2. 复制到当前环境
  3. 修改API密钥、路径等必要项
  4. 重启验证

预防措施

1. Git版本控制

cd ~/.openclaw
git init
git add config/
git commit -m "init config"

2. 定时自动备份

0 2 * * * cp ~/.openclaw/config/openclaw.json ~/.openclaw/backups/openclaw-$(date +\%Y-\%m-\%d).json

3. 验证后再重启

每次修改配置后，用 openclaw config validate 验证格式，再重启Gateway。90%的问题都能避免。

写在最后

配置文件坏了不可怕，可怕的是没有备份。赶紧检查一下你的备份策略吧。
